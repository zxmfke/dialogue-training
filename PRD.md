# 📋 PRD：咨询师 AI 陪练助手

## 1. 产品概述

### 1.1 产品定位
为医院咨询师（医美/口腔/体检科）提供 **轻量化 AI 话术陪练工具**，帮助咨询师快速掌握科室项目话术、应对患者异议、提升转化率，同时规避合规风险。

### 1.2 核心价值
- **碎片化学习**：3-5 分钟完成一次有效训练
- **场景化演练**：还原真实咨询对话，不是死记硬背
- **即时反馈**：练完立刻知道哪里说得好、哪里要改
- **合规护航**：自动检测违规词汇，降低投诉风险

### 1.3 目标用户
| 角色 | 痛点 | 使用场景 |
|-----|------|---------|
| 新入职咨询师 | 不懂项目怎么介绍，不敢开口 | 上岗前集中培训 |
| 经验咨询师 | 遇到难搞患者不知道怎么说 | 针对特定异议专项突破 |
| 科室主管 | 团队水平参差不齐，培训成本高 | 查看团队能力报告，定向辅导 |

## 2. 用户故事

### 2.1 咨询师侧
> 作为一名新入职的医美咨询师，我想在患者来之前快速练习一下"玻尿酸项目介绍"话术，这样接待时更有信心。

> 作为一名口腔科咨询师，当患者说"太贵了我要再考虑"时，我不知道怎么回应，想要看看销冠是怎么说的。

> 作为一名咨询师，我希望利用午休时间练 5 分钟，系统能给我推荐今天最适合练的内容。

### 2.2 管理侧
> 作为科室主管，我想知道团队里谁在"价格谈判"环节最薄弱，好安排针对性培训。

> 作为院长，我想确保咨询师的话术符合医疗广告法，避免违规风险。

## 3. 功能需求

### 3.1 核心功能模块

#### 模块 1：AI 陪练对话
| 功能点 | 描述 | 优先级 |
|-------|------|--------|
| 场景选择 | 按科室、项目、患者类型选择演练场景 | P0 |
| AI 患者角色 | 模拟不同性格患者（犹豫型、价格敏感型、冲动型） | P0 |
| 多轮对话 | 支持 5-10 轮真实对话流程 | P0 |
| 语音/文字输入 | 支持语音输入（还原真实咨询）和文字输入 | P0 |
| 中途提示 | 卡住时可查看提示，但不强制 | P1 |

#### 模块 2：话术评估与反馈
| 功能点 | 描述 | 优先级 |
|-------|------|--------|
| 综合评分 | 满分 100 分，维度：专业度、流畅度、转化率、合规性 | P0 |
| 逐句点评 | 标记出说得好的句子和需要改进的句子 | P0 |
| 优化建议 | AI 给出"更好的说法"参考 | P0 |
| 敏感词检测 | 自动标红违规词汇（如"最好""第一""根治"等） | P0 |
| 录音回放 | 可回听自己的语音，自我检视 | P1 |

#### 模块 3：话术库
| 功能点 | 描述 | 优先级 |
|-------|------|--------|
| 标准话术 | 各科室项目介绍标准话术 | P0 |
| 异议处理库 | 常见患者异议及应对话术 | P0 |
| 销冠话术 | 优秀咨询师的真实案例拆解（脱敏） | P1 |
| 话术搜索 | 按关键词快速查找话术 | P1 |

#### 模块 4：学习路径
| 功能点 | 描述 | 优先级 |
|-------|------|--------|
| 新手任务 | 新用户引导，完成 3 个基础场景 | P0 |
| 每日推荐 | 根据薄弱点智能推荐今日练习 | P1 |
| 专题训练 | 如"双 11 活动话术特训周" | P2 |
| 闯关模式 | 难度递增的挑战任务 | P2 |

#### 模块 5：数据看板（管理端）
| 功能点 | 描述 | 优先级 |
|-------|------|--------|
| 团队概览 | 团队平均分、练习次数、活跃人数 | P1 |
| 能力雷达图 | 各维度能力分布（专业度、异议处理、促成等） | P1 |
| 个人详情 | 单个咨询师的历史表现、成长曲线 | P1 |
| 薄弱点分析 | 团队共性薄弱环节，指导培训方向 | P2 |

## 4. 产品流程

### 4.1 主流程：完成一次陪练

```
打开小程序 → 选择科室/场景 → 查看场景背景 → 开始对话 
    ↓
AI 患者开场白 → 咨询师回复（语音/文字）→ AI 患者回应
    ↓
（循环多轮）→ 结束对话 → AI 评估报告 → 查看优化建议
    ↓
收藏话术 / 分享成绩 / 开始下一轮
```

### 4.2 关键交互细节

**场景选择页：**
- 顶部 Tab：医美科 / 口腔科 / 体检科 / 通用技能
- 每个场景卡片显示：场景名称、难度星级、预估时长、已练习人数
- 支持"快速开始"按钮（随机推荐一个场景）

**陪练对话页：**
- 顶部：当前场景名称 + 退出按钮
- 中部：对话气泡（患者左，咨询师右）
- 底部：语音输入按钮（按住说话）/ 文字输入切换
- 侧边：提示按钮（显示当前环节的关键话术要点）

**评估报告页：**
- 头部：综合得分 + 评级（S/A/B/C）
- 雷达图：4 个维度评分
- 对话回放：可展开查看完整对话
- 优化建议：针对低分环节给出具体改进话术
- 底部：再练一次 / 换个场景 / 分享

## 6. 技术要求

### 6.1 技术架构
- **前端**：微信小程序原生 / Taro 跨端框架
- **后端**：Node.js / Python FastAPI
- **AI 能力**：
  - 对话：GPT-4 / Claude / 国内大模型（合规考虑）
  - 语音：ASR（语音识别）、TTS（语音合成）
  - 评估：Prompt 工程 + 评分模型
- **数据库**：MongoDB（文档型，适合灵活的场景配置）
- **部署**：支持私有化部署（满足医院数据安全要求）

### 6.2 合规要求
| 合规项 | 方案 |
|-------|------|
| 医疗广告法 | 内置敏感词库，AI 生成内容自动过滤 |
| 数据安全 | 支持私有化部署，数据不出院 |
| 患者隐私 | 训练案例全部脱敏处理，禁止上传真实患者信息 |
| 等保要求 | 三级等保认证 |

### 6.3 性能要求
- 小程序首屏加载 < 2s
- AI 对话响应 < 3s
- 语音转文字 < 2s

## 9. 风险与应对

| 风险 | 应对方案 |
|-----|---------|
| 咨询师不爱用 | 降低使用门槛（3 分钟完成）、游戏化激励、主管强推 |
| AI 回答不准确 | 话术边界限定、敏感词过滤、人工审核兜底 |
| 医院数据安全顾虑 | 提供私有化部署方案、签署数据安全协议 |
| 内容更新跟不上 | 建立医院内容共创机制、销冠激励计划 |
